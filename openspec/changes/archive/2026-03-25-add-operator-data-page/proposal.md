## Why

当前桌面应用仍以窗口工具为主，缺少围绕《明日方舟》公开招募的核心业务页面，也没有一套可维护的干员基础数据来源。为了支撑后续标签推荐、干员筛选和数据维护能力，需要先在主界面中引入稳定的导航结构，并提供一个可以从公开来源抓取并展示干员数据的页面。

## What Changes

- 在主页面左侧增加导航菜单，明确区分不同业务页面，而不是继续把所有功能堆叠在单个视图中。
- 新增“干员数据”页面，用于展示当前已加载的干员数据列表和数据状态。
- 在“干员数据”页面新增“获取干员数据”操作，初始无数据时可由用户主动触发。
- 新增从 `https://wiki.biligame.com/arknights/公开招募工具` 抓取公开招募工具页面 HTML 并解析全部干员数据的能力。
- 将解析结果标准化为应用内可展示、可复用的干员数据结构，为后续公开招募规则匹配提供基础数据。

## Capabilities

### New Capabilities
- `app-navigation`: 主界面左侧导航菜单与页面切换能力。
- `operator-data-management`: 干员数据页面、数据抓取、HTML 解析和结果展示能力。

### Modified Capabilities
- None.

## Impact

- 影响 `frontend/src` 的应用布局，需要从单页窗口工具界面调整为带侧边导航的多页面结构。
- 影响 `main.go` 暴露的 Wails 接口，需要新增获取和解析干员数据的后端能力。
- 需要增加 HTML 抓取与解析相关依赖或封装，并定义干员数据模型与页面状态模型。
- 为后续公开招募推荐能力提供可复用的数据来源，降低后续规则实现时的数据准备成本。
