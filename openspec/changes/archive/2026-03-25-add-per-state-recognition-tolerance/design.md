## Context

当前识别设置已经具备“一个区域下多个状态图”的数据模型，每个状态图都绑定一个公开招募标签。之前如果只给模板或整个区域提供统一误差阈值，那么所有状态图都必须共享同一套容差规则，这会与现实场景冲突：某些状态图颜色稳定、应当严格匹配，另一些状态图因为高亮、阴影或轻微采样波动，需要更宽松的容差。用户已经明确纠正了这一点，因此本次设计必须把误差值和每一张识别图片状态绑定，而不是继续使用统一阈值。

这次改动会触及前端识别设置页面的状态表单、后端模板持久化结构、旧模板兼容逻辑和图片匹配算法。主要约束包括：
- 误差值必须跟状态图一起保存、读取和编辑，不能只存在模板级内存状态中。
- 每个状态图匹配时只使用自己的误差值，不受同一区域其他状态图配置影响。
- 旧模板没有该字段时必须稳定兼容，并保持默认严格匹配行为。
- 现有区域多状态匹配、公开招募标签返回和自动识别聚合流程不应因为误差值粒度变化而被重新设计。

## Goals / Non-Goals

**Goals:**
- 在识别设置页面为每个状态图提供独立误差值输入、展示和保存能力。
- 扩展状态图持久化结构，让每张状态图单独保存误差值。
- 在匹配时针对每张状态图使用自己的误差值执行逐像素绝对差值判断。
- 为旧模板提供默认误差值 `0` 的兼容加载逻辑。
- 补充测试，覆盖不同状态图不同容差、零容差和无效输入场景。

**Non-Goals:**
- 不恢复或保留模板级统一误差设置。
- 不引入新算法，如 OCR、平均误差评分、结构相似度或模糊匹配。
- 不改变区域坐标、标签绑定、多窗口识别或公开招募聚合契约。
- 不扩展到按颜色通道分别配置误差，当前每张状态图仍只使用一个整数阈值。

## Decisions

### 1. 误差值直接挂在每个区域状态图上
每个 `RecognitionRegionState` / `RecognitionRegionStateInput` 新增独立误差值字段；前端状态编辑列表中每一项都可单独填写该值，保存时与状态图一起落盘。

选择原因：
- 用户已经明确要求“每一个识别图片都有自己独立的误差设置”。
- 现有识别设置 UI 已经按状态图逐项展示标签和缩略图，把误差值放在同一层最符合用户心智模型。
- 状态图是实际参与比较的最小单元，误差值跟着它保存最自然。

备选方案：
- 保留模板级统一误差，再允许状态图覆盖。未采用，因为会引入两级优先级和额外心智负担。
- 把误差值挂在区域级。未采用，因为同一区域内不同状态图仍然可能需要不同容差。

### 2. 匹配时逐个状态图读取自己的误差值
区域匹配遍历状态图列表时，不再接收单一统一容差，而是每读取一个状态图就使用该状态图自己的误差值执行像素比较。状态图之间互不影响。

选择原因：
- 这能让同一区域内不同状态图在同一轮识别中使用不同严格度。
- 后端改动边界清晰，只需让状态匹配函数从状态对象取阈值，而不是新增更复杂的匹配调度逻辑。

备选方案：
- 先按区域或模板算一个有效容差，再传给状态图比较。未采用，因为会丢掉用户配置的独立粒度。

### 3. 旧状态图默认误差值为 0
读取旧模板时，如果状态图结构中不存在误差字段，则归一化为 `tolerance = 0`；新保存模板总是显式写出该字段。

选择原因：
- 旧模板必须继续可读且行为不变。
- 默认 0 对应严格完全匹配，是最稳定、最可预测的兼容策略。

备选方案：
- 默认给旧状态图一个非零误差。未采用，因为这会悄悄改变旧模板识别行为。

### 4. 前后端共同限制误差值为非负整数
前端对每个状态图误差值使用数值输入和保存前校验；后端在标准化输入时再次验证，防止负数或非法值进入模板文件。

选择原因：
- 误差值越界会直接改变匹配结果，必须双重兜底。
- 非负整数是最简单也最容易测试的输入约束。

备选方案：
- 只在前端校验。未采用，因为模板数据可能绕过 UI 进入后端。

## Risks / Trade-offs

- [状态图数量多时表单更密集] -> 让误差值与标签放在同一状态编辑卡片中，避免引入额外编辑层级。
- [不同状态图误差配置差异大，用户难以统一维护] -> 通过默认值 0 和明确说明文案减少误用，后续如有必要再补复制/批量设置能力。
- [旧模板没有独立误差字段] -> 读取时归一化为 0，新写入统一落新字段，避免兼容分叉。
- [匹配函数签名调整后影响现有测试与调用链] -> 保持区域级返回结构不变，只调整状态级比较入参和状态数据结构。

## Migration Plan

- 第一步：扩展状态图输入、状态图持久化结构和旧模板归一化逻辑，加入独立误差值字段。
- 第二步：调整状态图匹配函数，使用每张状态图自己的误差值执行图片比较。
- 第三步：在识别设置页面状态编辑区增加误差值输入和前端校验。
- 第四步：补充状态级误差值的保存、读取、兼容和匹配测试。
- 第五步：验证公开招募识别和区域匹配结果在不同状态图不同容差场景下仍正常工作。
- 回滚方式：移除状态图误差字段并恢复无容差或统一容差方案，同时保持旧模板可读。

## Open Questions

- 是否需要在 UI 中提供“复制上一张状态图误差值”之类的便捷操作，还是先只提供逐项手动输入。
- 误差值上限是否需要收紧到 255，还是先仅要求非负整数并在实现时酌情约束。
