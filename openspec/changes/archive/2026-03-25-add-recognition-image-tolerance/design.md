## Context

当前识别图片逻辑建立在严格逐像素相等比较之上，模板图和当前截图只要存在轻微颜色差异，就会直接被判定为不匹配。这种策略虽然简单，但在实际窗口截图、缩放采样、抗锯齿和录入模板截图的过程中，极易因为单个像素波动造成整张区域失配。现在需要把“完全相等”升级为“允许有限像素误差”的匹配方式，并让用户可以在识别设置页面配置这个容差值。

这次改动同时涉及前端识别设置表单、模板持久化结构、后端图片比较逻辑以及现有识别结果链路，因此需要先固定几个边界：
- 容差规则必须明确可测试，避免出现“看起来差不多”但无法解释的命中结果。
- 误差值需要随模板保存，而不是只存在页面临时状态中。
- 新匹配逻辑必须兼容现有区域状态匹配和公开招募自动识别，不改变后续区域聚合契约。
- 需要给未设置旧模板一个明确默认行为，避免旧模板在升级后不可读或全部误判。

## Goals / Non-Goals

**Goals:**
- 在识别设置页面新增图片匹配误差值配置，并让用户可以查看、编辑和保存该值。
- 将识别匹配逻辑改为基于模板图片与截图图片逐像素绝对差值比较，只要差值小于误差阈值就视为该像素通过。
- 把误差值持久化进识别模板，并为旧模板提供稳定默认值。
- 保持现有区域状态匹配、区域结果返回和公开招募自动识别链路可继续复用。
- 补充边界验证，避免负数、超大值或空值导致匹配逻辑失控。

**Non-Goals:**
- 不引入新的模板匹配算法，如 SSIM、特征点、OCR 或模糊模板匹配。
- 不改变识别模板的区域绘制、多状态标签或公开招募标签聚合逻辑。
- 不在本次中实现按颜色通道分别配置容差，先保持单一统一误差值。
- 不扩展为全局动态调参中心，本次仅在识别设置工作流内提供配置入口。

## Decisions

### 1. 误差值定义为单个像素通道差值上限，并使用统一整数阈值
每个像素比较时，对模板图与截图图的同一位置颜色分量取绝对差值；当参与比较的每个颜色分量差值都小于误差值时，该像素视为通过。误差值使用单一整数阈值，适用于整个状态图匹配，不拆分为 R/G/B/A 多个独立字段。

选择原因：
- 用户给出的规则是“模板图片减去截图图片的绝对值，小于误差值时视为匹配成功”，最直接的解释就是逐像素逐通道阈值判断。
- 单一整数阈值最容易让用户理解，也最容易做输入校验和测试覆盖。
- 保持统一阈值能把 UI 和持久化复杂度控制在最小范围。

备选方案：
- 使用整图平均误差作为阈值。未采用，因为会掩盖局部大偏差，且与用户描述不完全一致。
- 为每个颜色通道分别配置阈值。未采用，因为配置过重，当前没有明确需求。

### 2. 误差值作为模板级配置保存，并在区域状态匹配时统一应用
误差值挂在识别模板级别，而不是每个区域或每个状态单独配置。模板下所有区域状态匹配都复用同一个误差值。

选择原因：
- 识别设置页当前已经围绕“一个窗口模板对应多个区域”组织，误差值更适合作为模板整体匹配策略的一部分。
- 公开招募模板中的多个区域通常来自同一窗口截图和同一缩放环境，使用统一阈值更容易维护。
- 这可以降低 UI 密度，避免每个区域状态都增加重复配置。

备选方案：
- 按区域配置误差值。未采用，因为会显著增加编辑复杂度，而且当前需求并未要求细粒度差异化。
- 按状态图配置误差值。未采用，因为最难维护，也会使持久化结构膨胀。

### 3. 旧模板默认使用 0 误差值以保持向后兼容
读取未包含误差值字段的旧模板时，将其归一化为 `tolerance = 0`，即继续使用原来的严格完全匹配行为；新保存模板则始终显式写入误差值字段。

选择原因：
- 旧模板不应因结构升级而失效。
- 把默认值设为 0 可以确保旧模板行为不变，不会因为升级突然扩大匹配范围。
- 新旧模板在内存模型中统一后，匹配逻辑与 UI 都只需处理一种结构。

备选方案：
- 为旧模板默认设置一个非零容差。未采用，因为这会默默改变既有识别行为。

### 4. 识别设置页面提供受限数值输入，并展示默认/边界约束
页面为误差值提供明确的数值输入控件，限制为非负整数，并给出默认值与说明文案。保存前做前后端双重校验，防止无效值进入模板。

选择原因：
- 容差是直接影响匹配结果的核心参数，必须可见且可解释。
- 仅在后端兜底会让用户在保存前难以发现配置错误。
- 双重校验可以避免前端绕过或旧模板异常数据导致运行时问题。

备选方案：
- 隐藏误差值，仅使用固定默认值。未采用，因为用户明确要求在识别设置页面添加误差功能。

## Risks / Trade-offs

- [误差值设置过大导致误匹配] -> 通过非负整数约束、默认 0 和页面说明降低误用概率，并在测试中覆盖高容差场景。
- [模板级单一容差不适合所有区域] -> 先采用最简单可维护的模板级配置，若后续确有需要再扩展到区域级。
- [逐像素逐通道比较在大图上有一定开销] -> 当前区域图尺寸较小，且已有区域数量有限，性能风险可控。
- [旧模板未显式包含误差字段] -> 读取时归一化为 0，写回时统一落新字段，避免兼容分叉。

## Migration Plan

- 第一步：扩展识别模板结构，加入误差值字段和旧模板默认值归一化逻辑。
- 第二步：调整后端图片比较函数，按逐像素绝对差值和误差阈值执行匹配。
- 第三步：在识别设置页面增加误差值输入、展示和保存校验。
- 第四步：补充模板保存、模板读取、零容差、非零容差和边界值测试。
- 第五步：验证误差匹配对现有公开招募识别链路没有破坏。
- 回滚方式：移除误差值字段并恢复严格像素完全匹配，旧模板仍可按默认 0 容差读取。

## Open Questions

- 误差值的上限是否需要严格限定为 255，还是仅要求非负整数并允许后续根据实现再收紧。
- 页面是否需要实时预览“当前误差值下匹配是否通过”，还是先只在保存后用于正式识别。
